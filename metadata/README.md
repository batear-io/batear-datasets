# Metadata

Dataset manifest and labeling information.

## Files

| File | Description |
|---|---|
| `samples.json` | Master registry of all audio samples with sample rate, bit depth, timestamps, and tags |
| `audio_drone_distance_<DATE>.csv` | Per-WAV distance to one or more drones for a session (see below) |

## Drone-distance CSVs

Generated by `tools/drone_distance.py` from an AudioMoth `SUMMARY.CSV` (recorder
GPS comes from the GUANO `Loc Position` field, set via the AudioMoth Configuration
App's acoustic-chime feature) and the DJI image EXIF (per-frame GPS lat/lon/alt).

For each AudioMoth WAV the CSV reports, per drone:

- `<drone>_n` — number of EXIF samples falling inside the WAV's time window
- `<drone>_d3d_min/median/mean/max_m` — 3D distance recorder ↔ drone (Haversine + vertical)
- `<drone>_horiz_min/max_m` — horizontal distance only
- `<drone>_height_above_rec_min/max_m` — drone altitude minus take-off altitude minus recorder height

Plus `active_drone` (`mavic-pro`, `mini-4-pro`, `mavic-pro+mini-4-pro`, or `none`).

The per-drone take-off altitude is the GPS altitude of that drone's first photo;
this cancels the per-receiver GPS-altitude bias so heights are comparable across
drones recorded in the same session.

## Schema

Every entry in `samples.json` must include:

- `id` — unique identifier
- `filename` — relative path from repository root
- `category` — top-level class (`drone`, `bio`, `ambient`, `synthetic`)
- `sample_rate_hz` — original sample rate
- `bit_depth` — bits per sample
- `channels` — mono (1) or stereo (2)
- `duration_sec` — length in seconds
- `timestamp_utc` — ISO 8601 recording timestamp
